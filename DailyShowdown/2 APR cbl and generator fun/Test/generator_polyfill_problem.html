Assignment

Implement a generator function polyfill in TypeScript

TypeScript Intermediate Iterator protocol

Background

JavaScript's function* syntax and the yield keyword are handled by the compiler/runtime. In environments that don't support generators natively, we need to emulate their behaviour using plain functions and a state machine.

Your task is to implement a generator polyfill from scratch — without using function*, yield, or any generator-related built-ins.

Task

Implement the following in TypeScript:

Part 1 — Core types (10 marks)

Define the TypeScript interfaces for:

  • IteratorResult<T, TReturn> — the object returned by .next()
  • Generator<T, TReturn, TNext> — with .next(), .return(), .throw(), and [Symbol.iterator]()

Part 2 — State machine (20 marks)

Implement a createGenerator(steps) factory function that:

  • Tracks state across four phases: Created, Running, Suspended, Completed
  • Advances through steps on each .next() call
  • Passes the value from .next(v) back into the current step (two-way communication)
  • Throws an error if .next() is called while the generator is already running

Part 3 — Exit methods (10 marks)

Implement .return(value) and .throw(error) such that:

  • .return(v) immediately completes the generator and returns { value: v, done: true }
  • .throw(err) marks the generator as completed and re-throws the error
  • Any call to .next() after completion returns { value: undefined, done: true }

Part 4 — Iterable protocol (10 marks)

Ensure your generator object is iterable — it must work correctly with:

  • for...of loops
  • Spread syntax: [...gen]
  • Array destructuring: const [a, b] = gen

Constraints

  • You may not use function*, yield, or any generator built-in
  • You may not use async/await
  • All code must be valid TypeScript with strict mode enabled
  • No external libraries

Test cases

Your implementation must pass all of the following:

  1. A range generator makeRange(0, 3) yields 0, 1, 2 then returns done: true
  2. Calling .next(v) passes v into the current step
  3. Calling .return(42) mid-sequence gives { value: 42, done: true } and exhausts the generator
  4. for (const n of makeRange(0, 5)) iterates all five values without error
  5. Calling .next() on a completed generator always returns { value: undefined, done: true }

Hints

Think of a generator as a state machine with four states. Draw the transitions before you write any code.

Each "step" is a function that receives the value passed to .next() and returns an IteratorResult.

The [Symbol.iterator]() method should return this — the generator is both an iterator and an iterable.

Use a const enum for your state values — it keeps the code readable and compiles away at runtime.
